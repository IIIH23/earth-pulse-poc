# Stage 8 — Docs & Backup/Restore: Backup Inventory Spec

## Status: ✅ Complete

## Purpose

Define what to back up for the Pulse of Earth project, where it lives, how to restore it, and what is explicitly excluded (secrets, environments, generated artifacts).

## Backup Scope

### Tier 1 — Critical (must be in version control or backed up separately)

| Item | Location | Backup Method | Notes |
|------|----------|---------------|-------|
| Git repository | `/` (all tracked files) | Git remote (GitHub) | Primary source of truth. Push to GitHub = backup. |
| Python tools | `tools/*.py` | Git | All 10 tool modules + `__init__.py` |
| Tests | `tests/*.py`, `tests/*.sh` | Git | 14 test files + smoke_test.sh |
| Shell scripts | `scripts/*.sh`, `scripts/*.py` | Git | 11 shell scripts + 3 Python scripts |
| CI workflows | `.github/workflows/*.yml` | Git | ci.yml + smoke.yml |
| Documentation | `docs/*.md` | Git | 30+ spec and plan docs |
| Obsidian output | `docs/obsidian/*.md` | Git | Generated sync output |
| Infrastructure | `infrastructure/`, `compose/` | Git | Terraform, cloud-init, Docker Compose |
| Project config | `requirements-dev.txt`, `Dockerfile`, `compose.yaml`, `.gitignore` | Git | Build and dependency files |

### Tier 2 — Reproducible from code (no separate backup needed)

| Item | Location | Why not backed up |
|------|----------|-------------------|
| Python venv | `.venv/`, `.venv_autopilot/` | Reproducible via `python3 -m venv` + `pip install -r requirements-dev.txt` |
| `__pycache__` | Various dirs | Generated by Python at runtime |
| Pytest cache | `.pytest_cache/` | Generated by pytest |
| Autopilot logs | `logs/AUTOPILOT_LOG.md` | Git-tracked; historical record |
| Autopilot state | `AUTOPILOT_STATE.md` | Git-tracked |

### Tier 3 — Secrets (NEVER in Git, stored externally)

| Item | Storage | Access |
|------|---------|--------|
| `.env` | Owner's local only | Not tracked in Git |
| `TELEGRAM_BOT_TOKEN` | Environment variable | Set in shell profile or CI secrets |
| `TELEGRAM_CHAT_ID` | Environment variable | Set in shell profile or CI secrets |
| GitHub deploy key | `~/.ssh/github_ed25519` | Owner manages |
| VPS SSH keys | `~/.ssh/` | Owner manages |
| Cloudflare API token | Environment variable | Owner manages |
| Linear API token | Environment variable | Owner manages |

### Tier 4 — Generated runtime artifacts (excluded from backup)

| Item | Location | Why |
|------|----------|-----|
| Snapshot outputs | `/tmp/snapshots/` or configured path | Regenerated by `tools/snapshot.py` |
| Healthcheck reports | stdout/telegram/file | Regenerated by `tools/healthcheck_report.py` |
| Trend reports | stdout/file | Regenerated by `tools/trend.py` |
| Diff outputs | stdout/file | Regenerated by `tools/diff_snapshots.py` |

## Restore Procedure

### Full restore from Git

```bash
# 1. Clone the repository
git clone git@github.com:<org>/pulse-of-earth.git
cd pulse-of-earth

# 2. Create virtual environment
python3 -m venv .venv
source .venv/bin/activate

# 3. Install dependencies
pip install -r requirements-dev.txt

# 4. Verify tools compile
python3 -m py_compile tools/*.py

# 5. Run test suite
python3 -m pytest tests/ -q

# 6. Verify CI workflows present
ls .github/workflows/

# 7. Restore secrets (owner action)
# Copy .env from secure storage
# Set environment variables: TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID
```

### Partial restore (single tool)

```bash
# From the Git repo, any single file can be restored:
git checkout HEAD -- tools/healthcheck.py
git checkout HEAD -- tests/test_healthcheck.py
```

## Backup Commands

```bash
# Verify repository is clean and pushed
git status --porcelain --branch
git log --oneline origin/master..HEAD  # should be empty

# Create a local archive (excluding venvs and cache)
tar czf /tmp/pulse-of-earth-backup-$(date +%Y%m%d).tar.gz \
  --exclude='.venv*' \
  --exclude='__pycache__' \
  --exclude='.pytest_cache' \
  --exclude='*.pyc' \
  --exclude='.env*' \
  --exclude='secrets' \
  --exclude='credentials' \
  .

# Verify archive contents
tar tzf /tmp/pulse-of-earth-backup-*.tar.gz | head -20
```

## Backup Verification Checklist

After any restore or backup operation:

- [ ] `git status` shows clean working tree
- [ ] `python3 -m py_compile tools/*.py` succeeds for all tools
- [ ] `python3 -m pytest tests/ -q` passes (124+ tests)
- [ ] `python3 tools/inventory.py --help` runs without error
- [ ] `python3 tools/healthcheck.py --help` runs without error
- [ ] `.github/workflows/ci.yml` exists and is valid YAML
- [ ] `.env` is NOT present in the restored tree (secrets stay external)

## Decision Record

Date: 2026-06-29
Decision: Tiered backup strategy — Git is the primary backup for all code/docs/config. Secrets are never committed. Runtime artifacts are reproducible. Venvs are rebuilt from requirements file.
Rationale: Pulse of Earth is a code-and-docs project with no persistent runtime state (tools produce output on demand). The only irreplaceable assets are in Git. Everything else is either reproducible from code or managed externally as secrets.
